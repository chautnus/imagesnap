## Fixed

- **`sheets.ts` — `getSheetRows`**: No longer swallows 403/500/network errors as empty data. Only returns `[]` for 400/404 (genuinely missing sheet). Prevents auth failures from triggering default-category seeding.
- **`useAppData.ts` — `refreshData`**: Default category seeding now requires both `categories` AND `productNames` to be empty. Prevents overwriting a workspace where the user deleted all categories on purpose.
- **`StaffLogin.tsx`**: Now saves `masterSpreadsheetId` to `localStorage` and calls `POST /api/auth/session` after successful staff login, establishing the HTTP-only session cookie.
- **`session/route.ts`**: Session cookie payload now stores `masterSpreadsheetId` so it survives page refresh.
- **`useDashboardInit.ts` — `handleInit`**: Staff path now reads `masterSpreadsheetId` from session profile and stores it to localStorage, preventing `findOrCreateWorkspace()` from creating a blank sheet on a fresh device.

## Changed

- **`useDashboardInit.ts`** split from 298 lines → 173 lines
  - Extracted `useShareTarget.ts` (100 lines): IDB/PWA share-target state machine
  - Extracted `useSubStatus.ts` (44 lines): subscription status & quota fetching
  - Return API of `useDashboardInit` unchanged — no downstream impact on `page.tsx`
